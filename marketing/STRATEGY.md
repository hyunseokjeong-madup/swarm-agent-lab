# 마케팅 특화 에이전트 전략 (MADOBI)

## North Star
> **"숫자를 절대 틀리지 않는 마케팅 분석가."** 매드업(애드테크) 분석가의 루틴 업무를
> 정합성(reconciliation) 보장과 함께 자동화한다. 우리 핵심기술 = **검증 스웜**이 곧 마케팅의 통점
> (스프레드 요약·이벤트 분석에서 숫자 틀림)을 직접 해결한다.

## 왜 우리가 이긴다
- 일반 에이전트의 마케팅 분석 #1 불만 = **"숫자가 틀린다."** 우리는 모든 파생지표를 **원자료에서
  재계산 + 독립 교차검증 + 분해합=총계 검산**한다 (METRICS.md B항). 단일 루프(Hermes/Pi)엔 없는 안전장치.
- **놀리지 에셋화**: METRICS / RECON 규칙 / 소재 플레이북 / 리포트 템플릿을 재사용 자산으로 누적.
  쓸수록 계정·클라이언트별 맥락이 메모리에 쌓여 더 빨라지고 정확해진다(복리형).

## 자동화 대상 = 유저의 루틴 업무
| 업무 | 입력 | 산출(정합성 보장) |
|------|------|---------------------|
| 스프레드시트 요약 | CSV/시트 | 검산된 총계 + 핵심 파생지표 + ⚠경고 |
| 일/주/월 성과 리포트 | raw 지표 | 기간합 검산 + WoW/MoM + 코멘트 |
| 광고 소재(creative) 분석 | 소재별 지표 | 승자/패자(노출임계) + 피로 + 추천 |
| 이벤트/캠페인 분석 | 기간 지표 | pre/post 리프트 + 페이싱 + 이상치 |
| 채널/캠페인 정합성 점검 | 다차원 표 | 분해합=총계, 파생 역산 일치 검증 |
| 예산 페이싱/알람 | spend/budget | 소진율·초과예측·경고 |

## 처리 파이프라인 (검증 스웜 적용)
```
입력 파싱 ─▶ [Solver 스웜] 지표 재계산(3경로: 직접/역산/표준공식)
            ─▶ [Verifier 스웜] 정합성 불변식 검산(분해합·역산·가중평균·단위·반올림)
            ─▶ [Synthesizer] 검증 통과 수치만 보고 + ⚠불일치 표면화
            ─▶ [Knowledge] 이번 분석의 맥락·교훈을 메모리/자산에 기록
```

## 단계적 구현
1. **지식자산**: METRICS.md(완료) · RECON 체크리스트 · 소재 플레이북 · 리포트 템플릿.
2. **정합성 엔진**: `reconcile.py` — CSV의 분해합/파생역산/가중평균/단위 검산(실동작).
3. **스킬화**: `.claude/skills/marketing-analyst` — 유저가 호출하면 루틴 분석을 알아서 수행.
4. **메모리/히스토리**: 대화·분석 이력을 자동 파악·기록하는 프로토콜(MEMORY_PROTOCOL.md).
5. **하네스화(선택)**: 정기 리포트를 스케줄/훅으로 자동 실행.
